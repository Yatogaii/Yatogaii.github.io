吃透LeetCode-1207

发表于 2019-10-17 | 分类于 LeetCode ， Easy

题目

给你一个整数数组 arr，请你帮忙统计数组中每个数的出现次数。

如果每个数的出现次数都是独一无二的，就返回 true；否则返回 false。

示例 1：

输入：arr = [1,2,2,1,1,3]
输出：true
解释：在该数组中，1 出现了 3 次，2 出现了 2 次，3 只出现了 1 次。没有两个数的出现次数相同。

示例 2：

输入：arr = [1,2]
输出：false

示例 3：

输入：arr = [-3,0,1,-3,1,1,1,-3,10,0]
输出：true

提示：

1 <= arr.length <= 1000
-1000 <= arr[i] <= 1000

思路

这里我想到的思路是很基本的思路，就是先遍历一遍数组，然后新建一个Map，存放数组中所有元素出现的个数。

之后在利用Set，对Map进行遍历，如果有重复的元素Set.contains()==True就返回False，否则循环结束说明没有重复的，返回True

进阶求解

进阶思路

先对数组进行排序，由于数组中肯定会存在重复的元素，所以排序后的数组重复的数肯定是扎堆出现的，这时通过计算最左边出现的数l和最右边出现的数r的位置之差+1，即可算出每个元素出现的个数。

每计算出一个出现次数，直接add进Set，如果有重复，直接返回，这样在某种情况下不需要遍历出全部的数组，更加高效。

总结

看到这种题的常规思路是遍历数组，然后计算各个元素出现的次数，之后利用Set或一些其他的容器来判断是否有相同的元素。

但是这种方法在第一次遍历数组的同时就计算出了每个元素出现的个数，在整体上都变得更加高效了。
